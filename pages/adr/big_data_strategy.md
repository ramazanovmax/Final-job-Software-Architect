## ADR-009: Стратегия обработки больших данных: Слоистая архитектура (Raw → Aggregated → Analytics)

**Статус**: Принято

**Контекст**: Огромный объём сырых данных трекинга (GPS-треки, показатели датчиков) и необходимость быстрой аналитики для бизнеса.

**Решение**:
* Сырые события -> Kafka -> Object Storage (S3) как Data Lake.
* Агрегированные данные (минута/час/день) -> ClickHouse для оперативной аналитики.
* Бизнес-метрики -> PostgreSQL для дашбордов.

**Последствия**: Возможность строить сложные аналитические отчёты и ML-модели. Высокие затраты на ETL-пайплайны и хранение.
