## ADR-008: Интеграция с E-commerce: Асинхронный Event-Driven Bridge с Circuit Breaker

**Статус**: Принято

**Контекст**: Существующие системы e-commerce могут быть нестабильны. Их сбои не должны влиять на core-функции приложения (тренировки).

**Решение**: Создать отдельный интеграционный сервис EcommerceBridge. Коммуникацию строить через события (Kafka). Для синхронных вызовов к e-commerce API использовать паттерн Circuit Breaker. Кешировать каталог товаров.

**Последствия**: Высокая отказоустойчивость. Данные о наличии товаров могут быть не совсем актуальными (eventual consistency), что приемлемо для нашего сценария.
