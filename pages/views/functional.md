## a. Функциональное представление\

**Цель**: Показать, какие функции предоставляет система и как они связаны.

**Для кого**: Product Owner, бизнес-аналитики, тестировщики.

**Основные компоненты и их ответственность**:
1. Модуль управления пользователями (User Management):
* Регистрация, аутентификация (через Auth Service).
* Управление профилем (цели, виды спорта, уровень).
* Настройки приватности и согласий (Consent Management).
2. Модуль тренировок (Workout Core):
* Запись тренировки (старт/пауза/финиш).
* Сбор данных с датчиков (GPS, пульс).
* Базовый анализ и сохранение результатов.
* Сравнение с предыдущими результатами.
3. Модуль инвентаря (Gear Inventory):
* Добавление/удаление экипировки (вручную или из e-commerce).
* Расчет износа на основе данных тренировок.
* Формирование события GearWearThresholdReached.
4. Модуль социального графа (Social Graph):
* Управление связями (друзья, подписки).
* Управление группами (создание, вступление).
* Формирование базовой социальной ленты (последние активности друзей).
5. Сервис уведомлений (Notification Service):
* Обработка событий из Kafka (WorkoutCompleted, GearWearThresholdReached).
* Отправка push, email, SMS по шаблонам.
6. Интеграционный слой (E-commerce Bridge):
* Кеширование каталога товаров.
* Синхронизация покупок пользователя.
* Инициирование процесса покупки (deep link с SSO).

**Ключевые связи**: Пользователь через UI вызывает операции модулей. Модули взаимодействуют через синхронные вызовы внутри монолита и публикуют события в Kafka для асинхронных процессов (уведомления, аналитика).
