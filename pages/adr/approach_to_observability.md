## ADR-004: Подход к наблюдаемости: Единый стек на основе OpenTelemetry

**Статус**: Принято

**Контекст**: Микросервисная система требует глубокой наблюдаемости для отладки и мониторинга SLA.

**Решение**: Принять OpenTelemetry как стандарт для инструментации всех сервисов. Использовать:

* **Prometheus + Grafana** — для метрик и алертов.
* **Grafana Tempo / Jaeger (через OTLP)** — для распределённой трассировки.
* **Loki / ELK Stack** — для централизованного логирования.

**Последствия**: Единый источник данных для диагностики. Требует внедрения стандартов логирования и инструментации в каждый сервис с первого дня.
